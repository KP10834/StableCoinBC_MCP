현재 브랜치의 변경사항을 자동으로 테스트해줘.

## 동작

`qa_test_branch` MCP 도구를 사용하여 현재 브랜치를 자동 테스트한다.

## 절차

### 1단계: 브랜치 테스트 실행

$ARGUMENTS 가 있으면 옵션으로 파싱해서 적용해:
- `--skip-build` → `skipBuild: true` (빌드 건너뛰기)
- `--keep-alive` → `keepAlive: true` (테스트 후 서비스 유지)
- `--all` → `testAll: true` (변경 핸들러 + 전체 회귀 테스트)
- `--base <branch>` → `base: "<branch>"` (비교 기준 브랜치 지정)
- `--timeout <ms>` → `timeout: <ms>` (응답 대기 시간)
- 인자 없으면 → 기본값으로 실행 (부모 브랜치 자동 감지)

`qa_test_branch` 도구를 호출해. 이 도구가 다음을 자동으로 수행한다:
1. git diff로 변경 파일 감지
2. 영향받는 핸들러 자동 식별
3. 빌드
4. 서비스 시작 (PM2)
5. 인프라 연결 확인 (Kafka, Redis, Bundler, RPC, DB, Health)
6. 변경된 핸들러 Kafka 테스트 (메시지 발행 → 응답 대기 → 검증)
7. 서비스 중지

### 2단계: 결과 해석

테스트 결과를 보고 아래 기준으로 판단해:

| 결과 | 조치 |
|------|------|
| 전체 PASS | "테스트 통과" 보고 |
| 일부 FAIL | 실패한 핸들러의 에러 원인 분석 후 수정 방향 제안 |
| 빌드 실패 | 빌드 에러 로그 분석 후 수정 방향 제안 |
| 인프라 연결 실패 | 어떤 인프라가 안 되는지 알려주고 환경 확인 안내 |

## 사용 예시

- `/qa` — 현재 브랜치 변경 핸들러만 테스트
- `/qa --all` — 전체 회귀 테스트 포함
- `/qa --skip-build` — 빌드 건너뛰고 테스트만
- `/qa --base dev` — dev 브랜치 기준으로 비교
- `/qa --keep-alive` — 테스트 후 서비스 유지 (수동 확인용)
